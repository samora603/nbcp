# RFCs (Requests for Comments)

RFCs are design proposals discussed **before** an irreversible Architecture Decision Record is accepted.

Use RFCs when:

- Multiple viable approaches exist
- The change spans several modules or teams
- Trade-offs need explicit comparison

## Process

1. Copy [TEMPLATE.md](TEMPLATE.md) to `NNNN-short-title.md`.
2. Open a PR for discussion.
3. After consensus, promote the outcome to an ADR (accept/reject/supersede).
4. Link the ADR back to the RFC.
